# マルチテナント（会社IDログイン）セットアップ

1つのシステム上で複数会社を運用するための手順です。

## やること（必須）

1. Supabase SQL Editor で `supabase/multi_tenant.sql` を実行する
2. Vercel / `.env.local` に `SUPABASE_SERVICE_ROLE_KEY` があることを確認する
3. アプリを再デプロイする
4. ログイン画面で次のように入る

| 項目 | 例（自社） |
|------|------------|
| 会社ID | `tokito` |
| ログインID | 従来のメールアドレス（例: `tokito@tokito-co.jp`） |
| パスワード | 従来どおり |

既存ユーザーは SQL 実行時に自動で会社 `tokito` へ紐付けされます。  
ログインIDは当面 **メールアドレス全体** です。

## 新しい短いログインIDを発行する場合

1. `/admin`（システム管理者: `tokito@tokito-co.jp`）を開く
2. 「ログインID付きユーザーを発行」で作成
3. 利用者には `会社ID=tokito` + 発行したログインID + パスワード を伝える

## 他社を追加する場合（商用化）

SQL 例:

```sql
INSERT INTO companies (company_code, name, allow_legacy_email_login)
VALUES ('acme', '株式会社ACME', false);
```

その後、管理者向け API `/api/admin/company-users` で
`companyCode: "acme"` のユーザーを発行します。

データは会社ごとに分離されます（案件JSON・スケジュール・工程会議・監査ログ・バックアップ）。

## ログイン方式の変更点

- 公開の新規登録はログイン画面から削除
- 認証は `/api/auth/login` 経由（メールアドレスは画面に出さない）
- 新規ユーザーの内部メールは `ログインID@会社ID.genka.local`
