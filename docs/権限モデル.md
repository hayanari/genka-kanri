# 権限モデル（商用）

## 役割

| 役割 | 範囲 | できること |
|------|------|------------|
| **システムオーナー** | 全会社 | 全社のユーザー発行・権限変更・削除、会社横断のアカウント管理 |
| **会社オーナー / 会社管理者** | 自社のみ | 自社ユーザーの発行・権限変更・削除 |
| **入力可 (editor)** | 自社データ | 案件・スケジュール等の編集 |
| **閲覧のみ (viewer)** | 自社データ | 閲覧のみ |

システムオーナーは `platform_owners` テーブルで管理します（初期: `tokito@tokito-co.jp`）。

会社内権限は `company_users.role` です。

## セットアップ

1. Supabase SQL Editor で `supabase/platform_roles.sql` を実行
2. 各社の管理者を任命:

```sql
UPDATE company_users cu
SET role = 'owner'
FROM companies c
WHERE cu.company_id = c.id
  AND c.company_code = 'gesuikanri'
  AND lower(cu.auth_email) = 'ここに下水管理者のメール';
```

3. デプロイ後、管理者でログインするとサイドバーに「アカウント管理」が出ます
