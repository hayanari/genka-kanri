# 下水管理興業の統合手順

目的: 別 Vercel / 別 Supabase だった「下水管理興業」を、本番（`app genka kanri` + `genka-kanri.vercel.app`）の会社 `gesuikanri` に統合する。

## 前提（確定済み）

- 本番 Supabase: **app genka kanri**
- 会社コード: **gesuikanri**
- `gesuikanri.vercel.app` は残さない

## 手順

### 1. 本番 SQL

Supabase（**app genka kanri**）の SQL Editor で実行:

1. `supabase/add_company_gesuikanri.sql`

### 2. データ移行

1. プロジェクト直下に `.env.migrate.local` を作成（**Git にコミットしない**）:

```
SOURCE_SUPABASE_URL=（genka kanri gesuikanri の Project URL）
SOURCE_SUPABASE_SERVICE_ROLE_KEY=（同上の service_role）
TARGET_SUPABASE_URL=（app genka kanri の Project URL）
TARGET_SUPABASE_SERVICE_ROLE_KEY=（同上の service_role）
MIGRATE_DEFAULT_PASSWORD=好きな初期パスワード
```

キーの場所: Supabase → Project Settings → API

2. ローカルで実行:

```bash
node scripts/migrate-gesuikanri.mjs
```

3. ログに出たユーザー一覧を確認（新規作成された人は初期パスワード使用）

### 3. ログイン確認

1. `https://genka-kanri.vercel.app` を開く
2. 会社ID `gesuikanri` + メール + パスワード
3. 下水側の案件・スケジュールが見えること
4. 会社ID `tokito` で時任データが崩れていないこと

### 4. Vercel の整理

1. Vercel プロジェクト **`gesuikanri`** を開く
2. Settings → Git → **Disconnect**
3. 動作確認後 Settings → General → **Delete Project**

（ドメインは残さない方針のため、追加設定は不要）

### 5. 旧 Supabase の整理（余裕があれば）

1. 移行後1〜2週間は **genka kanri gesuikanri** を停止せず残す（保険）
2. 問題なければ Pause / Delete

## 注意

- 移行スクリプトは再実行すると、gesuikanri 会社のスケジュール等を消し入れてやり直します
- 既存 auth ユーザーが本番側に既にあるメールの場合は、そのアカウントを会社 `gesuikanri` に紐付けます（パスワードは本番側のまま）
